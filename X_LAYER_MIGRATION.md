# Mimir × X Layer — migration log & deploy guide

Mimir was originally an AI-settled prediction market on **Arc (Circle L1)**. It has been migrated to **X Layer Testnet** (OKX zkEVM L2, chain id `1952`). Stakes are now an ERC-20 stablecoin (`USDC_TEST`, 6 decimals). Gas is paid in native **OKB** (18 decimals).

## What changed

| Area              | Before                              | After                                              |
| ----------------- | ----------------------------------- | -------------------------------------------------- |
| Chain             | Arc Testnet (5042002)               | X Layer Testnet (1952)                             |
| Native gas        | USDC (Circle native, 18 dec)        | OKB (18 dec)                                       |
| Stake token       | USDC, used via `msg.value`          | USDC_TEST ERC-20 at `0xcB8BF24c...c79D` (6 dec)    |
| Min stake         | 2 USDC                              | 1 USDC                                             |
| Stake mechanism   | `msg.value` (native)                | `approve` + `transferFrom` (ERC-20)                |
| Agent signer      | Circle W3S (Programmable Wallets)   | Local private key via viem                         |
| Cross-chain bridge| CCTP V2 + `/bridge` page            | Removed — users on-ramp from OKX directly          |
| Unified balance   | Circle Gateway widget               | Removed                                            |
| Explorer          | testnet.arcscan.app                 | www.oklink.com/xlayer-test                         |
| Faucet            | faucet.circle.com                   | www.okx.com/xlayer/faucet                          |

## Backward-compat shims (intentional)

These keep older import paths compiling without a project-wide rename:

- `lib/arc.ts` → re-exports everything from `lib/xlayer.ts`
- `xLayerTestnet` is also exported as `arcTestnet`
- `OKB_DECIMALS` / `OKB_UNIT` are also exported as `USDC_DECIMALS` / `USDC_UNIT` (note: the legacy "USDC" decimals constant now refers to OKB; the actual ERC-20 USDC stake token uses its own 6-decimal accounting in `lib/contract.ts`)
- `okbToWei` / `weiToOkb` / `formatOkb` are also exported as `usdcToMicro` / `microToUsdc` / `formatUsdc`
- `executeContract(...)` in `lib/circle-w3s.ts` keeps the same signature the agents were already calling — internally it now uses viem + a private key resolved from `ORACLE_PRIVATE_KEY` / `CREATOR_PRIVATE_KEY`, falling back to `wallets.local.json` in dev

New code should import from `lib/xlayer.ts` and use the OKB-named functions for gas-side values and explicit 6-decimal USDC handling for stake-side values.

## Deploy flow

A few X Layer-specific gotchas matter here:

- **Chain id is `1952`, not `195`.** Some older docs and tutorials use `195` for X Layer Testnet; the live one we deployed against is `1952`. This caught us out — wagmi was silently signing for the wrong chain until we set it correctly in `lib/xlayer.ts`.
- **X Layer Testnet rejects EIP-1559 transactions.** Both `deploy/deploy.ts` and the agent signer in `lib/circle-w3s.ts` (`signAndSendLegacy`) build and sign **legacy EIP-155** transactions with explicit `gasPrice`. Do not use viem's default `sendTransaction` flow — it sends an EIP-1559 envelope that the RPC will reject.
- **Use the alternate RPC.** The default `https://testrpc.xlayer.tech` works for most reads, but writes were flaky during our deploy. The alternate `https://xlayertestrpc.okx.com` is what we successfully deployed against and what the agents currently use.

```bash
# 1. Install deps
npm install

# 2. Generate wallets (writes wallets.local.json — git-ignored)
npm run wallets:generate

# 3. Fund the three addresses on X Layer Testnet
#    Faucet: https://www.okx.com/xlayer/faucet  (request OKB + USDC for each)
#    Verify: npx tsx scripts/check-balances.ts

# 4. Compile Mimir.sol (solc 0.8.28 viaIR; outputs artifacts/Mimir.bin + abi)
npm run contract:compile

# 5. Deploy — signs a legacy EIP-155 tx against xlayertestrpc.okx.com
DEPLOYER_PRIVATE_KEY=$(jq -r .deployer.privateKey wallets.local.json) \
ORACLE_ADDRESS=$(jq -r .oracle.address wallets.local.json) \
npm run deploy:contract

# 6. Paste into .env.local:
#    NEXT_PUBLIC_CONTRACT_ADDRESS=0x0924af6f439ff8da91d209733ed16b8ad7c8ce53
#    NEXT_PUBLIC_DEPLOY_BLOCK=30762544
#    ORACLE_PRIVATE_KEY=0x...
#    CREATOR_PRIVATE_KEY=0x...
#    GEMINI_API_KEY=...

# 7. Run the dev server and the two agents
npm run dev               # Next.js frontend on :3000
npm run workers           # oracle + market-creator concurrently
```

## How USDC stake amounts work

USDC_TEST has **6 decimals**. The contract holds the stake amount in 6-decimal units:

- 1 USDC = `1_000_000` (the `MIN_STAKE` constant)
- 2 USDC = `2_000_000`
- 10 USDC = `10_000_000`

Every stake-bearing write is a two-step transaction:

```text
1. usdc.approve(NEXT_PUBLIC_CONTRACT_ADDRESS, stakeAmount)
2. mimir.createClaim(...)   // contract calls transferFrom(msg.sender, this, stakeAmount)
```

The contract reverts if `allowance < stakeAmount`. The frontend, the oracle's auto-challenger, and the market-creator all perform this two-step approve flow.

OKB on EVM has 18 decimals just like ETH. It is only used to pay gas — never as a stake.

## Known follow-ups

- `lib/contract.ts` still has a handful of `@deprecated not used on X Layer` JSDoc comments inherited from the Arc era. Not user-visible; left for the next cleanup pass.
- `lib/circle-w3s.ts` keeps its old filename to avoid a project-wide rename. It's a viem signer now, not a Circle W3S client. Worth renaming when there's a quiet PR slot.
- The legacy-named env knobs `CHALLENGE_STAKE_OKB` and `CREATOR_STAKE_OKB` actually drive USDC stake sizes in the agents (they configure the 6-decimal amount). Rename to `*_USDC` in a future pass.
- `lib/xlayer.ts` exposes `USDC_DECIMALS = 18` for back-compat; the real stake-token decimals (6) are handled separately in `lib/contract.ts`. Consolidate when the back-compat shims are removed.
- A second RPC endpoint as a hot-fallback would be useful — `xlayertestrpc.okx.com` has gone briefly unavailable a few times during testing.
